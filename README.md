# TornaMap - Documentación

Repositorio centralizado de documentación para el proyecto **TornaMap**.

## ¿Qué es TornaMap?

TornaMap es un proyecto compuesto por un backend y un frontend. Este repositorio centraliza toda la documentación técnica del proyecto para facilitar el desarrollo, mantenimiento y colaboración entre los integrantes del equipo.

## Estructura de la documentación

La documentación está organizada en las siguientes secciones:

| Sección | Descripción |
|---------|-------------|
| [Backend](docs/backend/README.md) | Documentación del servidor, API REST, modelos de datos y lógica de negocio |
| [Frontend](docs/frontend/README.md) | Documentación de la interfaz de usuario, componentes y flujos de navegación |

## Wiki

La documentación detallada del proyecto se encuentra en la [Wiki de este repositorio](../../wiki). La Wiki está organizada en las siguientes secciones principales:

- **Backend** — arquitectura del servidor, endpoints de la API, base de datos y guías de instalación.
- **Frontend** — estructura de componentes, rutas, decisiones de diseño y guías de uso.

## Convenciones

- Toda la documentación debe estar escrita en **español**.
- Los archivos de documentación se guardan en formato Markdown (`.md`).
- Cada sección principal tiene su propio `README.md` como punto de entrada.

## Contribución

Para agregar o modificar documentación:

1. Crear una nueva rama a partir de `main`.
2. Realizar los cambios en los archivos correspondientes.
3. Abrir un Pull Request describiendo los cambios realizados.
